# Dashboard de Performance de Vendas

## Objetivo do Projeto
O objetivo deste projeto foi o desenvolvimento de um dashboard de vendas focado na conversão de dados brutos em inteligência de negócio. A solução visa proporcionar uma visualização clara e organizada de indicadores de desempenho, facilitando a análise de tendências e o suporte estratégico para a tomada de decisão.

---

## Estrutura da Solução
O projeto foi estruturado em quatro camadas fundamentais para garantir a organização e escalabilidade dos dados:

* **Bases:** Camada de dados brutos contendo o histórico de transações e registros de vendas.
* **Calculos:** Camada de processamento onde são realizadas as agregações, fórmulas e tabelas dinâmicas que alimentam os componentes visuais.
* **Dashboard:** Interface principal de visualização, contendo gráficos interativos, cartões de KPI e filtros para análise de dados.
* **Assets:** Repositório de elementos visuais e componentes de design utilizados para otimizar a experiência do usuário (UX).

---

## Indicadores de Desempenho (KPIs)
O dashboard foi projetado para monitorar as seguintes métricas principais:

1. **Volume de Vendas:** Quantidade total de itens comercializados no período analisado.
2. **Faturamento Bruto:** Receita total gerada pelas transações.
3. **Analise de Sazonalidade:** Comportamento das vendas ao longo do tempo para identificação de picos e quedas.
4. **Desempenho por Segmento:** Distribuição de resultados por categorias de produtos ou vendedores.

---

## Tecnologias e Metodologias
* **Ferramenta de Analise:** Microsoft Excel.
* **Processamento:** Utilização de tabelas dinâmicas e funções avançadas para automação da aba de cálculos.
* **Visual Data Storytelling:** Design de interface focado na clareza de informações e facilidade de leitura.
* **Organizacao de Dados:** Separação entre base de dados e camada de apresentação para evitar corrupção de fórmulas.

---

## Instrucoes de Uso
1. **Atualizacao de Dados:** Novos registros devem ser inseridos na aba Bases seguindo o padrão de colunas existente.
2. **Processamento:** Após inserir novos dados, as tabelas na aba Calculos devem ser atualizadas para refletir as mudanças.
3. **Navegacao:** O usuário final deve interagir exclusivamente com a aba Dashboard, utilizando os filtros disponíveis para segmentar as análises.
